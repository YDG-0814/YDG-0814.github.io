SpringBoot项目常用依赖

一、springboot项目常见的pom.xml文件的配置依赖

二、父依赖

只用来作为其他 maven项目的 parent，因此它除了 pom文件并没有什么有效的内容

三、SpringBoot核心启动器

包含各种springboot的配置日志等，创建项目时会自动引入该依赖

注解@controller、@Service、@Component、@Resource 是spring的，所以spring boot创建完成后就可以使用

四、Web依赖

用于web开发场景，包含了 RESTful和Spring MVC，并且默认使用了内置的Tomcat服务器。

支持注解：@RestController、@RequestMapping、@ResponseBody

五、Lombok依赖

lombok最优秀的就是注解了，一个注解就干掉了很多代码

实体类中的注解.

@Data ：直接可以省略了Get、Set方法

@Slf4j ：不需要单独引入日志依赖和配置日志，直接log.info() 打印日志

六、Thymeleaf依赖

thymeleaf 是一个很强大的视图解析工具

SpringBoot支持的thymeleaf页面模板技术，thymeleaf支持 h:text规则

默认存放模板页面的路径在src/main/resources/templates 或者 src/main/view/templates

默认的页面文件后缀是.html

七、Mybatis依赖

数据处理层持久层框架，连接数据库

八、Mybatis-plus依赖

在mybatis基础上的升级版工具，避免了使用mybatis时需要编写大量的xml文件

九、MySQL驱动依赖

配置mysql依赖时，不写版本号xx.xx.xx的话，就会引入mysql依赖的默认版本

SpringBoot2.1.x以后默认使用的是mysql 8版本，

SpringBoot2.1.x之前默认使用的是mysql 5.x版本

在配置数据源的时候，就有差异

十、Test测试依赖

用于测试，提供了多个测试库，包括 JUnit Jupiter、Hamcrest 和 Mockito。

十一、Fastjson 依赖

Fastjson是一个 Java 库，可以将 Java 对象转换为 JSON 格式，当然它也可以将 JSON 字符串转换为 Java 对象。

十二、springboot热部署

修改java代码后，不用重启项目就能直接最新测试，省略了不断修改代码不断重启项目的麻烦

十三、JWT依赖

十四、aop依赖

使用 Spring AOP和AspectJ的面向切面编程场景。

十五、commons-lang3依赖

commons-lang3是一个小而全的Java工具类库，类里面有很多对时间、数组、数字、字符串等的处理。

十六、log依赖

十七、mail和email依赖

十八、redis依赖
